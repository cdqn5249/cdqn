## II. The Exact Ledger Mathematical Framework

Before specifying the high-level program, we analyze the structural and physical frictions of legacy financial accounting systems and how the $Qn$ primitive resolves them [4].

### A. Bypassing Floating-Point Non-Associativity
As analyzed in Chapter 1.1, floating-point addition and multiplication are non-associative [4]. When performing sequential currency conversions across asynchronous nodes, this non-associativity introduces microscopic rounding errors [4]. Let $A$, $B$, and $C$ be three distinct currency types (e.g., USD, EUR, VND). We evaluate the transaction of converting 1 unit of $A$ to $B$, then $B$ to $C$, versus converting $A$ directly to $C$. Under classical floating-point arithmetic, the sequential operations accumulate rounding drift [4]:

$$(1.0 \times \text{Rate}_{A \to B}) \times \text{Rate}_{B \to C} \quad \neq \quad 1.0 \times \text{Rate}_{A \to C}$$

This rounding discrepancy violates the SMR determinism constraint [7]. Replicas will diverge in their numerical states based purely on the order of thread execution, creating a vector for state-machine divergence, which must be secured against instruction-timing leaks and speculative side-channel attacks at the physical register level [4, 7, 68, 69].

### B. The 2-adic and 5-adic Split Register Model
To resolve this representation loss, the $Qn$ primitive performs exact base translations [8]. The number 10 is composite, meaning the 10-adic absolute value does not define a multiplicative norm on its own [8]. However, under the algebraic properties of profinite rings, the ring of $10$-adic integers $\mathbb{Z}_{10}$ decomposes exactly into the product of the $2$-adic and $5$-adic integers [67]:

$$\mathbb{Z}_{10} \cong \mathbb{Z}_2 \times \mathbb{Z}_5$$

By extending this to their fields of fractions, we obtain the coprime decomposition [8]:

$$\mathbb{Q}_{10} \cong \mathbb{Q}_2 \times \mathbb{Q}_5$$

When a user inputs a decimal fraction (e.g., $0.1_{10} = 1/10$), the $QnLang$ compiler splits the value into two parallel registers [8]. The **$2$-adic component** is executed natively on standard binary CPU registers for maximum processing speed [2]. The **$5$-adic component** (representing the fractional portion that cannot be represented exactly in binary) is carried algebraically as an exact, symbolic cofactor (the remainder $r_z$) [8]. This non-destructive remainder carriage is managed as a **single-owner linear-typed thunk ($\multimap$)**, ensuring that the physical register holds the exact, un-truncated mathematical value of the currency balance [9].

## VI. Glossary of Terms for Non-Technical Readers

*   **Symmetric Monoidal Category (SMC):** A mathematical structure where calculations are represented as diagrams of parallel wires (representing data states) and boxes (representing operations), allowing parallel and sequential steps to be compiled without a traditional program counter.
*   **Data-Oriented Programming (DOP):** A programming paradigm that separates code (behavior) from data (state), representing data as simple, immutable, and generic structures to maximize CPU cache performance and systems modularity.
*   **Phantom Type:** A type-system parameter that is enforced and checked statically by the compiler during type-analysis, but is completely erased (removed) during code generation so that it consumes zero runtime memory or storage.
*   **Symmetric Monoidal Stack Invariant:** A compile-time constraint enforcing that every stack register representing a linear resource must be read exactly once, preventing memory leaks and use-after-free exploits.
*   **Manual Translation Validation:** An early-stage compiler research methodology where a high-level program is manually translated into its target intermediate representation by human developers to verify the mathematical and operational semantics of the runtime before the automated compiler frontend is written.
