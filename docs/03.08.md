### A. The Separation of Data and Behavior

1.  **Nouns (Data Inputs):** Mapped to the base wire type $n$, representing primitive types (such as `i64_z`) [10, 27].
2.  **Transitive Verbs (Morphisms):** Mapped to the complex adjoint type $n^r \otimes s \otimes n^l$, representing directed transition boxes (such as `add2` or `mul2`) [10, 27].

This pregroup mapping is designed to ensure that nouns are compiled strictly as un-scaled input wires on the stack, and verbs are compiled strictly as directed transition morphisms, enforcing Data-Oriented Programming (DOP) natively at the compiler frontend [3, 9, 10].

## IV. Packed ASCII Integer Arrays & Branch-Free Constant-Time Tokenizing

At the physical gate level, the local processor is specified to execute string and keyword matching without exposing private data to microarchitectural state leaks [13, 21].

### A. The 1-Byte Fixed-Width ASCII Constraint
Grounded in O. Aciicmez et al. (2007) and E. Bangerter et al. (2011), variable-length multi-byte character decoding and string parsing on standard processor architectures are highly susceptible to microarchitectural timing and branch-predictor side-channel leaks [33, 34].

To neutralize this timing leak, the $QnLang_{0.1}$ seed compiler strictly enforces **fixed-width, 1-byte ASCII** for its packed string representations on the stack [02.03.md, 03.07.md]:
*   **The Invariant:** Every character occupies exactly 1 byte, allowing the string-matching and lexical analysis to execute as a sequence of flat, branch-free, and data-oblivious `i64` comparison operations [02.04.md, 03.07.md].
*   **The Compilation:** This completely bypasses the branch predictor unit. The host JIT-compiler lowers these operations to native conditional moves (`CMOVcc`), designed to guarantee that physical registers execute string matching in a constant, data-oblivious number of clock cycles [02.04.md].

### B. Lowering Multi-Character Tokens to Branch-Free `select` and `br_table` Instructions
To compile multi-character tokens (such as `<-`, `//`, `>>`) without generating conditional branch instructions, the compiler's backend lowers the comparisons to WebAssembly's native branch-free **`select`** and table-driven **`br_table`** instructions [2, 13, 29].
1.  **The Character Lookahead:** When the lexer reads a character (such as `<` ASCII 60), it is specified to immediately dispatch an asynchronous lookahead call to read the next character (`-` ASCII 45) [1, 9].
2.  **The Table-Driven Dispatch:** To completely eliminate sequential, nested `if/else` jumps, the main parser loop compiles token matching strictly as an **array-lookup `br_table` jump table [29].**
3.  The compiler initializes a lookup table in Slab B, mapping the ASCII codes directly to consecutive branch indices [02.06.md]. The parser loads the index and executes `br_table` in constant time [29]. At the CPU level, this lowers directly to an indirect jump or branchless conditional moves, **completely bypassing the conditional branch predictor unit** and ensuring that execution time is hypothesized to remain perfectly identical and flat-line regardless of the input data [29].

### C. The $\mathbb{Q}_2$ Complex Root Deadlock and Field Clamping Invariant
The CDQN stack decomposes base 10 exactly as $\mathbb{Q}_{10} \cong \mathbb{Q}_2 \times \mathbb{Q}_5$ to execute native binary register math [01.4.md, 01.12.md]. Standard algebraic papers assume the imaginary unit $i = \sqrt{-1}$ is represented as a standard, convergent $p$-adic integer using Hensel's Lemma over $p \equiv 1 \pmod 4$ (such as $...212_5$ in $\mathbb{Q}_5$) [01.4.md, 01.12.md].

    Under standard $2$-adic number theory, the polynomial $x^2 + 1 = 0$ has **no roots** inside the field of $2$-adic numbers $\mathbb{Q}_2$ [6, 7].

To resolve this mathematical gap without generating dynamic compiler crashes or requiring complex, hardware-dependent quadratic field extensions ($\mathbb{Q}_2(i)$) that would double register pressure and violate Wasm stack-only invariants [02.03.md], **this specification proposes to resolve complex roots in-place outside the compiler core via a specialized CDQN package (`ql_complex`) [03.05.md, 03.07.md].** `[Empirical Horizon]`
*   The `ql_complex` package defines a multi-dimensional coordinate tuple as parallel monoidal wires ($A \otimes A$) and encapsulates the exact 2-adic phase arithmetic within a dedicated, copyleft-compliant algebraic library [02.02.md, 03.05.md].
*   The compiler core remains strictly flat and un-extended [02.02.md], while the library maps the complex coordinates natively, allowing standard, un-hardened CPU registers to execute exact phase rotations on-the-fly [02.04.md, 03.07.md].

## V. Thread Scheduler and Token Reordering Mechanics

To execute and verify parallel, multi-core execution threads safely without register-level lock overhead, the reordering mechanics of the **Qn Thread Scheduler** are specified [10, 11].

```
                     THE QN THREAD SCHEDULER
    [ Token Stream Input: n9 n0 v h ]
                 │
                 ▼
    [ TIER 1: Dependencies ] ──► Ordered by poset restriction maps ρ_U,V [10]
                 │               - Static DAG prevents race conditions
                 ▼
    [ TIER 2: Constraints ] ───► Verified via coboundary checks dx = 0 [13]
                 │               - Blocks compilation of invalid transitions
                 ▼
    [ TIER 3: Capabilities ] ──► Mapped via cohesive Resource Functor Φ [15]
                                 - Clamps scale if memory exceeds hardware limits
```

The scheduler is compiled as a priority queue $\mathcal{H}_{\text{stack}}$ designed to statically and dynamically reorganize pending execution threads strictly by a three-tier mathematical hierarchy [01.9.md]:

$$\mathcal{H}_{\text{stack}} = \text{Queue}\big( \text{Tier 1: Dependencies} \quad \longrightarrow \quad \text{Tier 2: Constraints} \quad \longrightarrow \quad \text{Tier 3: Capabilities} \big)$$

### A. Tier 1: Dependencies (Priority 1)
Dependencies are modeled strictly as morphisms (restriction maps $\rho_{U, V}$) in our context poset category $\mathcal{C}$ [10]. If task $B$ depends on the output of task $A$, there exists an exact restriction mapping [10]:

$$\rho_{A,B}: Q(A) \to Q(B)$$

The scheduler reorganizes the queue so that the stalk $Q(A)$ is evaluated and stabilized before the evaluation of $Q(B)$ can begin [10, 11]. This defines a strictly ordered, directed acyclic graph (DAG) of dependencies directly within the type system, designed to prevent race conditions natively [10, 50 in 02.14].

### B. Tier 2: Constraints (Priority 2)
Once the dependency order is established, the scheduler evaluates the security and correctness constraints. In the $Qn$ model, safety is verified by checking the local coboundary equation [13, 39 in 02.02]:

$$dx = 0 \quad \text{and} \quad H^1(X, \mathcal{F}) = 0$$

If a pending thread attempts an execution step that introduces a non-zero coboundary ($dx \neq 0$), representing an unauthorized capability escalation or a memory overflow, **the scheduler blocks the thread before it can allocate physical registers**, aiming to eliminate speculative leaks [13].

### C. Tier 3: Capabilities (Priority 3)
The finalized, safe dependency chain is mapped to the physical hardware site $\mathcal{H}$ using the resource-cohesive functor [15]:

$$\Phi: \mathcal{C} \to \mathcal{H}$$

The functor $\Phi$ measures the active capabilities of the local SoC (such as cache capacities and register bus widths) [15]. Following linear logic, physical memory allocation is treated as a consumable, non-duplicable linear asset [15]. If the physical memory required to run the current chain exceeds the hardware bounds:

$$\text{Memory}\big(Q(U)\big) > \text{Capacity}\big(\Phi(U)\big)$$

the compiler automatically uses the profinite truncation functor $\tau_z$ to reduce the Zoom level $z$ to a shallower depth $z' < z$, shrinking the memory footprint until the task size is guaranteed to fit within the physical memory limits before execution, designed to prevent Out-of-Memory crashes [6, 15].

### D. The Deterministic Scheduling Invariant (Side-Channel Mitigation)
Grounded in *Deterministic-by-default scheduling for side-channel mitigation* (2023), to prevent the scheduler's dynamic reordering based on dependency resolution from introducing new timing side-channels, execution schedules are designed to be *deterministic-by-default* [15]. The scheduler is specified to execute identical, padded queue transitions regardless of data value states, attempting to prevent thread-scheduling latencies from leaking private registers [12, 15].

Standard category papers assume a strictly abelian category ($\mathbf{CondAb}$) and linear boundary relations ($d^1 \circ d^0 = 0$) computed in polynomial time [01.2.md, 02.11.md]. They omit the fact that physical gauge fields (such as gravitational structures or non-abelian Yang-Mills fields) are governed by **non-abelian gauge groups** (like $SU(2)$ or $SU(3)$) where non-linear curvature invariants cannot be computed as sparse linear systems.

```wat
    call $fd_read
    local.set $unused

    i32.const 12
    i32.load
    i32.const 1
    i32.ne
    if
      i32.const -1
      return
    end

    i32.const 8
    i32.load8_u
  )

  ;; 8. The terminal, branch-free constant-time Soft Halt operator (shalt)
  (func $shalt (param $numerator i64)
    (local $status i32)
    
    i32.const 0
    i32.const 1
    local.get $numerator
    i64.eqz
    select
    local.set $status
    
    local.get $status
    call $proc_exit
  )

  ;; 9. Main Tokenizer Loop
  (func (export "_start")
    (local $char i32)
    (local $next_char i32)
    (local $sp i32)
    (local $index i32)
    (local $i i32)

    i32.const 0
    local.set $sp

    ;; Initialize the branch table indices in Slab B (offsets 1024 to 1152)
    ;; Default index is 5 (error/default)
    i32.const 0
    local.set $i
    (loop $init_loop
      local.get $i
      i32.const 128
      i32.lt_s
      if
        ;; Calculate address: 1024 + i
        i32.const 1024
        local.get $i
        i32.add
        i32.const 5       ;; Default index is 5 (error)
        i32.store8
        
        local.get $i
        i32.const 1
        i32.add
        local.set $i
        br $init_loop
      end
    )
    
    ;; Set explicit indices:
    ;; Offset 1024 + 60 ('<'): store index 0 (bind)
    i32.const 1084 ;; 1024 + 60
    i32.const 0
    i32.store8

    ;; Offset 1024 + 47 ('/'): store index 1 (parallel)
    i32.const 1071 ;; 1024 + 47
    i32.const 1
    i32.store8

    ;; Offset 1024 + 62 ('>'): store index 2 (pipe)
    i32.const 1086 ;; 1024 + 62
    i32.const 2
    i32.store8

    ;; Offset 1024 + 104 ('h'): store index 3 (halt)
    i32.const 1128 ;; 1024 + 104
    i32.const 3
    i32.store8

    ;; Offset 1024 + 48 to 57 ('0' to '9'): store index 4 (digit)
    i32.const 48
    local.set $i
    (loop $digit_init
      local.get $i
      i32.const 58
      i32.lt_s
      if
        i32.const 1024
        local.get $i
        i32.add
        i32.const 4
        i32.store8
        
        local.get $i
        i32.const 1
        i32.add
        local.set $i
        br $digit_init
      end
    )

    (loop $token_loop
      call $read_char
      local.set $char

      ;; Check for EOF
      local.get $char
      i32.const -1
      i32.eq
      if
        i32.const 1
        local.get $sp
        call $exit_with_scrub
      end

      ;; Skip whitespace: Space (32), LF (10), CR (13)
      local.get $char
      i32.const 32
      i32.eq
      local.get $char
      i32.const 10
      i32.eq
      i32.or
      local.get $char
      i32.const 13
      i32.eq
      i32.or
      if
        br $token_loop
      end

      ;; Boundary check: prevent out-of-bounds table read (ASCII 0 to 127)
      local.get $char
      i32.const 127
      i32.gt_s
      if
        i32.const 1
        local.get $sp
        call $exit_with_scrub
      end

      ;; Load branch table index from Slab B
      local.get $char
      i32.const 1024
      i32.add
      i32.load8_u
      local.set $index

      ;; --- THE CONSTANT-TIME TABLE-DRIVEN DISPATCH ENGINE ---
      block $default
        block $digit
          block $halt
            block $pipe
              block $parallel
                block $bind
                  local.get $index
                  br_table $bind $parallel $pipe $halt $digit $default
                end
                
                ;; Case 0: bind "<-"
                call $read_char
                local.set $next_char
                
                ;; Note: This error/trapping 'if' block is an acceptable side-channel risk because
                ;; it leads immediately to $exit_with_scrub (unconditional process termination).
                local.get $next_char
                i32.const 45
                i32.ne
                if
                  i32.const 1
                  local.get $sp
                  call $exit_with_scrub
                end
                i64.const 10      ;; TOKEN_BIND
                local.get $sp
                call $push
                local.set $sp
                br $token_loop
              end
              
              ;; Case 1: parallel "//"
              call $read_char
              local.set $next_char
              
              ;; Note: This error/trapping 'if' block is an acceptable side-channel risk because
              ;; it leads immediately to $exit_with_scrub (unconditional process termination).
              local.get $next_char
              i32.const 47
              i32.ne
              if
                i32.const 1
                local.get $sp
                call $exit_with_scrub
              end
              i64.const 11      ;; TOKEN_PARALLEL
              local.get $sp
              call $push
              local.set $sp
              br $token_loop
            end
            
            ;; Case 2: pipe ">>"
            call $read_char
            local.set $next_char
            
            ;; Note: This error/trapping 'if' block is an acceptable side-channel risk because
            ;; it leads immediately to $exit_with_scrub (unconditional process termination).
            local.get $next_char
            i32.const 62
            i32.ne
            if
              i32.const 1
              local.get $sp
              call $exit_with_scrub
            end
            i64.const 12      ;; TOKEN_PIPE
            local.get $sp
            call $push
            local.set $sp
            br $token_loop
          end
          
          ;; Case 3: halt "h"
          i64.const 13      ;; TOKEN_HALT
          local.get $sp
          call $push
          local.set $sp
          local.get $sp
          call $pop_val
          i64.const 13
          i64.sub
          call $shalt
        end
        
        ;; Case 4: digit "0-9"
        i64.const 14      ;; TOKEN_DIGIT
        local.get $sp
        call $push
        local.set $sp
        local.get $char
        i32.const 48
        i32.sub
        i64.extend_i32_u
        local.get $sp
        call $push
        local.set $sp
        br $token_loop
      end
      
      ;; Case 5: default / error
      i32.const 1
      local.get $sp
      call $exit_with_scrub
    )
  )
)
```
