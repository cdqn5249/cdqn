Quang Number (Qn) Foundational Axioms

This document defines the formal axioms of the Quang Number (Qn). Every first-class entity manipulated by the CDQN ecosystem is represented by one or more Quang Numbers. These axioms constitute the formal foundation of the entire CDQN stack.

Research Question

Can numbers abstract anything?

This research question motivates the development of the Quang Number formalism. It is intentionally not an axiom.

First Principle (FP0)

Every first-class entity manipulated inside the CDQN formalism is represented as one or more Quang Numbers.

Examples include

  • primitive numbers
  • operators
  • semantics
  • mathematical objects
  • expressions
  • persistent knowledge
  • reuse knowledge
  • intermediate representations
  • compiler objects
  • runtime objects

Immediate Consequences

  • Finite dependency graphs
  • Finite validation
  • Finite persistence
  • Finite compilation
  • Finite execution
  • Deterministic reproducibility
  • Immutable provenance

Turing Completeness

The CDQN formalism intentionally adopts Finite Constructive Computation. Consequently, QnLang is intentionally not Turing-complete. This property is derived from QA0 and is therefore considered a theorem instead of a foundational axiom.

Scope

These axioms define only the ontology of Quang Numbers. They do not define

  • Q0-Q9
  • operators
  • language grammar
  • compiler implementation
  • runtime implementation
  • distributed execution

Those specifications derive from the present document.
