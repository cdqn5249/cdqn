CDQN Documentation Standard

Purpose

Documentation SHALL be human-readable, AI-readable and form the project graph.

Document Object Definition

  • Unique identity
  • Dependencies
  • Constraints
  • Capabilities
  • Version
  • Lineage

Required Metadata

qn:id qn:type qn:version qn:status qn:author

HTML Profile

header nav main article section footer

Documentation Graph

Every HTML document is a node. Every hyperlink is a typed edge.

Hyperlinks

dependency derived-from implements references prev next

Normative Rules

  • MUST use HTML5
  • MUST include metadata
  • MUST join the documentation graph
  • SHOULD use semantic DOM
  • MAY use JSON-LD/RDFa
