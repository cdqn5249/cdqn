Purpose

This document defines the canonical documentation standard for the Chained and Distributed Quang Numbers (CDQN) project.

The objective is to ensure that every concept within the CDQN ecosystem is documented consistently for both humans and AI agents.

Documentation Invariant 001

Every concept shall have exactly one canonical semantic HTML document, one stable identifier, one permanent URL, and one structured metadata block.

Best of Standards (BoS)

CDQN adopts the Best of Standards (BoS) principle.

Whenever an open, mature, and widely adopted standard satisfies a requirement, CDQN shall reuse that standard instead of introducing a new one.

Innovation is reserved for problems that existing standards cannot adequately solve.

Canonical Documentation Format

The canonical documentation format of CDQN is semantic HTML5.

Markdown, PDF, presentations, generated summaries, and other formats are considered derived representations.

Only the canonical HTML document defines the authoritative version of a concept.

One Concept — One Document

Each semantic concept shall be documented by one HTML document only.

A document may reference many concepts, but it shall define only one canonical concept.

Canonical Structure

Every documentation page should follow the same logical organization.

  1. Document Information
  2. Purpose
  3. Definition
  4. Motivation
  5. Formal Specification
  6. Relationships
  7. Examples
  8. References
  9. Version History

Stable Identifiers

Every concept shall possess one unique identifier within the CDQN knowledge graph.

Identifiers should remain stable throughout the lifetime of the project.

Examples include:

  • CDQN
  • CDQN.QN
  • CDQN.QM
  • CDQN.QS
  • CDQN.QNIR
  • CDQN.QNLANG

Permanent URLs

Each concept shall be published through one canonical URL.

The repository and GitHub Pages together establish the permanent publication path of every document.

Humans and AI Agents

The documentation is intentionally designed to be readable by both humans and AI agents.

Semantic HTML provides a shared representation that can later be extended using structured metadata while remaining fully compatible with current web technologies.

Future Extensions

Future versions of this standard may define:

  • JSON-LD metadata
  • Concept lineage
  • Cross-reference graphs
  • Version lineage
  • QPolicy integration
  • QGovernance integration
  • Machine-readable identifiers
