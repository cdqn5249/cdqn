### 1. Introduction: The Geometry of the Primitive

In legacy systems, to grant a number directional meaning (positive/negative), architectures traditionally sacrifice a magnitude bit (e.g., the Two's Complement sign bit). To grant a number spatial meaning (e.g., 3D coordinates or high-dimensional cryptographic lattices), architectures rely on disparate data structures that require extensive serialization when migrating between non-volatile storage and active memory.

**[Empirical Horizon]** The CDQN framework posits that vector direction and dimensionality are not intrinsic properties of scalar magnitude, but properties of *spatial context*. By theoretically offloading polarity and dimension to the Operational Set ($Op$) and treating the Maths Set ($M$) purely as an absolute scalar distance, the Qn primitive aims to natively adopt the properties of a geometric vector space. This document formalizes the rules proposed to govern this spatial behavior.

### 2. Topological Polarity: The Origin of Charge

#### 2.1 The Bottleneck of Legacy Status Registers
**[Scientific Consensus]** In modern high-performance superscalar processors, relying on centralized hardware status registers (such as the FLAGS register in x86 or CPSR in ARM) creates strict instruction dependencies. Updating a Carry/Borrow flag after arithmetic operations introduces pipeline bottlenecks. Architectures such as RISC-V intentionally omit condition flags, relying instead on direct compare-and-branch instructions to maximize Out-of-Order (OoO) execution throughput.
* **Source 1:** *Waterman, A., & Asanović, K. (2020). "The RISC-V Instruction Set Manual."*
* **Source 2:** *Hennessy, J. L., & Patterson, D. A. (2024). "Computer Architecture: A Quantitative Approach."*

#### 2.2 The Compare-and-Set Polarity Protocol
**[Empirical Horizon]** To pursue Hardware Agnosticism across diverse instruction set architectures, the Qn primitive proposes determining vector charge via the stateless **Compare-and-Set Polarity Protocol**, avoiding standard ALU underflow panics and status-flag dependencies.

1. **Unsigned Comparison:** When executing a subtraction that may cross zero ($M_A - M_B$), the architecture intends to utilize a native unsigned comparison instruction (e.g., `sltu` in RISC-V). This outputs a pure boolean state ($1$ if boundary crossed, $0$ otherwise).
2. **Flag Binding:** This boolean output is theoretically routed directly into the `[Neg]` polarity bit within the 32-bit $Op$ set.
3. **Branchless Absolute Difference:** Utilizing bitwise masking derived from the boolean state, the hardware computes the absolute difference. If the boundary is crossed, a native two's complement inversion restores the magnitude to an absolute positive value.

**[Open Implementation Challenge]:** While the mathematical logic of branchless compare-and-set operations is sound, mapping these explicit routing behaviors to optimal microcode or compiler intrinsic functions across both x86 and RISC-V backends without re-introducing pipeline stalls is deferred to Series 03 (`cdqnLang` specifications).

### 3. Dimensionality ($d$) and Tensor Scaling

**[Scientific Consensus]** The efficient execution of multidimensional data requires strict memory contiguity to leverage Single Instruction, Multiple Data (SIMD) hardware. Furthermore, memory-safe data reinterpretation requires strict compiler aliasing compliance to prevent undefined behavior during aggressive optimization.
* **Source 1:** *Arpaci-Dusseau, R. H., & Arpaci-Dusseau, A. C. (2018). "Operating Systems: Three Easy Pieces."* (Regarding extent-based allocation).
* **Source 2:** *ISO/IEC 9899:2018 (C17 Standard), Section 6.5/7* & *Sutter, H., & Alexandrescu, A. (2004).* (Establishing the invalidity of legacy union-based type punning).

#### 3.1 The Contiguous Extent Protocol
**[Empirical Horizon]** To prevent duplicating the 384-bit Semantic Set ($S$) across multi-dimensional arrays, the architecture proposes the **Contiguous Extent Protocol**. When $d > 1$, the tensor is structured into two roles:
1. **The Header Qn (`Q_Head`):** The first 64-byte block. The $S$ set holds the Semantic Hypervector hash, and the $Op$ set contains the dimensionality flag (e.g., `[d=256]`).
2. **The Extent Qns (`Q_Ext`):** Subsequent contiguous 64-byte blocks. The compiler is intended to reinterpret the raw 512 bits as exactly **eight 64-bit absolute integers ($M$)**. To guarantee compiler alias safety across `cdqnLang` backends, this reinterpretation explicitly mandates strict-aliasing-compliant bit-casting (e.g., `std::bit_cast`).

#### 3.2 Arbitrary Dimensions ($Nd$) and Partial RAM Projection
Because dimensions are proposed to be stored as contiguous integers, the architecture theoretically possesses $O(1)$ deterministic locality. If a mathematical operation requires evaluating only dimension 500 of a $10,000d$ tensor, the OS calculates the exact byte offset and fetches only the specific 64-byte Extent Qn into RAM. 

**[Open Implementation Challenge]:** Managing physical disk fragmentation and Flash Translation Layer (FTL) behaviors that may disrupt the physical contiguity of `Q_Ext` chains on modern NVMe drives is an open OS-level challenge. The specific file-system mechanics necessary to guarantee this contiguous allocation will be addressed in Series 03 and Series 04 testing.

### 4. The Ontological Taxonomy (Classes of Qns)

To prevent the semantic hallucinations endemic to probabilistic models, the CDQN architecture theorizes a strict epistemological hierarchy, classifying Qns based on the **Fundamental Theorem of Arithmetic**.

#### 4.1 The `[Mag]` vs `[Dom]` Operational Flags
**[Empirical Horizon]** The architecture posits explicit $Op$ set flags to separate scalar quantities from semantic categories:
* **`[Mag]` (Magnitude):** Instructs the ALU to treat $M$ strictly as a scalar quantity (e.g., counting physical units).
* **`[Dom]` (Domain):** Instructs the system that $M$ is a Topological Coordinate. The scalar quantity is ignored, and the integer designates semantic lineage via prime factorization.

#### 4.2 Anchor Qns (`pQn`) and Structural Symbiosis
**[Scientific Consensus]** The Fundamental Theorem of Arithmetic dictates that every integer greater than 1 is uniquely factored into prime numbers. Additionally, Vector Symbolic Architectures can utilize superposition to bundle concepts deterministically.
* **Source 1:** *Hardy, G. H., & Wright, E. M. (2008). "An Introduction to the Theory of Numbers."*
* **Source 2:** *Plate, T. A. (1995). "Holographic Reduced Representations."*

**[Empirical Horizon]** When the machine maps novel domains of human knowledge (e.g., "Biology"), it aims to assign them to available Prime Numbers (e.g., `pQ11`) flagged as `[Dom]`. A composite coordinate like `Q[Dom](143)` mathematically factors into $11 \times 13$, deterministically proving the concept is a topological intersection of the `pQ11` and `pQ13` domains.
* **Symbiotic Classes:** **Twin Primes (`twinQn`)** naturally anchor Coupled Dualities. **Perfect Numbers (`perfQn`)** anchor Equilibrium States.

**[Open Implementation Challenge]:** Runtime prime factorization of massive integers introduces severe latency overhead, conflicting with the $O(1)$ speed objective. While potential solutions involve "Small Math" offset calculations (Aperiodic Routing) to locate pre-calculated intersections on disk without brute-force computation, the exact graph-traversal algorithms and disk storage formats are explicitly deferred to Paper 01.4.

#### 4.3 Fractional Qns and VSA Bundling
Subjective human concepts (e.g., linguistic nuances) are proposed to map dynamically as Fractional Qns drifting between established `pQn` domains via the Contextual Zoom ($z$). To support internationalization and absolute synonyms (e.g., "Zero" vs "Nullity"), the architecture posits utilizing VSA **Bundling (Superposition)** to create a language-agnostic hypervector representation independent of ASCII dictionaries.
