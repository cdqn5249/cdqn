## II. Architectural Specifications of the Stage 2 Testbed

We specify the resource-management, capability, and parsing architectures of the Stage 2 testbed, demonstrating how the compiler maps abstract invariants directly to physical stack instructions [2, 10].

### A. The Bounded $p$-adic Slab Allocator and Capability Shield
Grounded in Bala Subramanyan (2025) and Robert N. M. Watson et al. (2015), we compartmentalize the flat WebAssembly linear memory heap into disjoint, nested **$p$-adic memory balls** to prevent out-of-bounds corruption [14, 19, 65]. 

The `C2_ledger_thermal_test.wat` source code partitions the memory array into two distinct, hardware-aligned memory slabs [02.06.md]:
1.  **Slab A (Primary - Offset 16 to 1024):** Allocated exclusively to store the local, memory-isolated data stack of the primary execution thread [02.05.md, 02.06.md].
2.  **Slab B (Secondary - Offset 1024 to 2048):** Allocated exclusively to store the speculative, asynchronous $5$-adic background remainder thunks, preventing cross-core cache invalidations [02.06.md, 02.06a.md].

We propose to represent the memory pointers strictly as **opaque reference handles** (simulated via an index pointer local variable `local $hp i32` mapping to private Wasm Tables) [02.03.md, 02.06.md]. 

To prevent guest programs from faking system-calls, the compiler's backend lowers the abstract algebra morphisms strictly to WebAssembly’s native, sandboxed table structures, sealing the WASI `fd_write` import directly into the first slot of the table [19, 20]. All system-level dispatches are compiled strictly as **Table-Lookup `call_indirect` instructions** [19, 65].

### B. The Symmetric Monoidal Parsing Engine and Effect Rows
Grounded in Joachim Lambek (2008) and Bob Coecke et al. (2010), the parser reads the sequential input tokens on-the-fly and reduces them using pregroup monoidal contractions [10, 27 in 02.05]. 
*   **The Contraction:** Wires are compiled as Single-Writer Wires [14 in 02.03], and grammatical reductions are computed purely as monoidal contractions (cup-like wires), directly constructing the flat $QnIR_2$ hypergraph dataflow with zero heap allocation [22, 26 in 02.02].

*   **The Effect Checker:** The compiler's type-checker statically verifies the capability effect rows (`@ wasi_io`) and information-flow control labels (`#constant_time`) [02.08.md, 03.03.md]. Any un-declared or un-handled side effect is flagged as a compile-time capability violation, blocking code generation [15 in 02.03, 52 in 03.01].

      local.get $sp

## VI. Glossary of Terms for Non-Technical Readers

We define the foundational concepts used in this specification to ensure interdisciplinary readability [1].

### A. Substructural and Allocator Terminology
*   **$p$-adic Tree-Structured Slab Allocator:** A memory management model that partitions the flat WebAssembly linear memory heap into disjoint, nested $p$-adic memory balls, providing spatial capability bounds checking natively.
*   **Single-Writer Wire:** A security constraint where every data channel has exactly one producer and one consumer, preventing data-race hazards across parallel processor cores.
*   **Host-Level Affine Teardown:** A security recovery protocol where the host operating system or kernel catches guest-level traps, walks the capability tables, and manually zeroes out speculative memory segments before recycling the sandbox to prevent data-remanence leaks.
