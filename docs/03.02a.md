## II. Mathematical Formulation of the Pregroup Grammar Parser

We specify the algebraic mechanics of how the parsing engine translates high-level natural language tokens into strict, target-independent SMC string diagrams [10, 27].

### A. The Pregroup Lexicon Mapping
A pregroup is a partially ordered monoid where every element $t$ has a left adjoint $t^l$ and a right adjoint $t^r$ satisfying the contraction inequalities [27]:

$$t \otimes t^l \le 1, \quad t^r \otimes t \le 1$$

We define the lexicon mapping in $QnLang_1$ to assign pregroup types to each word in a sentence [27]:
1.  **Nouns (Data Inputs):** Mapped to the base wire type $n$ representing primitive types (e.g. `i64_z`) [10, 27].
2.  **Transitive Verbs (Morphisms):** Mapped to the complex adjoint type $n^r \otimes s \otimes n^l$, where $s$ represents the sentence start type [10, 27].

This lexicon mapping separates data from behavior, ensuring that nouns are compiled strictly as un-scaled input wires and verbs are compiled strictly as directed transition morphisms [10, 52].

### B. Grammatical Reduction as Monoidal Contraction
Let $W = [w_1, w_2, \dots, w_k]$ be a parsed sequence of words with assigned pregroup types $T = [T_1, T_2, \dots, T_k]$ [27]. The parser computes the grammatical reduction of the sentence by applying the contraction inequalities sequentially from left to right [27]:

$$\text{Reduction}: T_1 \otimes T_2 \otimes \dots \otimes T_k \to \mathbf{1}$$

Under the category-theoretic string diagram calculus, these contractions are represented strictly as **cup-like wires** that connect the output of one wire directly to the input of another, completely eliminating the need for variable scope tracking or program counter registers [22, 26]:

```
                     THE MONOIDAL CONTRACTION
              Word 1 (n) ───┐   ┌─── Word 2 (n^r)
                             \ /  (Cup wire contraction: n ⊗ n^r -> 1)
                              X
                             / \
              Word 3 (s) ───┘   └─── Terminal Unit 1
```

We hypothesize that this pregroup reduction allows the parser to construct the flat $QnIR_1$ hypergraph in linear time relative to the sentence length [26, 27]. `[Empirical Horizon]` 

If the reduction fails to contract to the terminal unit type $\mathbf{1}$, the compiler's parser rejects the program at compile-time as a syntax-error, preventing un-grammatical or un-typeable statements from ever generating bytecode [2, 23].

We specify how the parsing engine integrates the dual-nature Zoom scale ($z$) to manage quantitative metric precision and qualitative structural domains concurrently [01.3_oom_audit.md].

### B. Quantitative Zoom ($z_{\text{quant}}$) and Monadic Scale Alignment
If the parsed statement contains numerical magnitude variables, the parser uses the **Quantitative Zoom scale ($z_{\text{quant}} \in \mathbb{N}$)** to enforce metric exactness [01.3_oom_audit.md, 01.4.md]:

To prevent unconstrained semantic drift and "numerology" without carrying metadata to the registers, the compiler enforces the Concentric Semantic Rings strictly at the type-checking boundary [01.7.md].

    *   *Structural Failure:* If the compiler cannot distinguish semantic domains, dynamic "numerology" or capability escalation attacks can corrupt the core registers, violating the "Blind Register" and absolute security invariants [01.9.md, 02.04.md].

The parser type-checker statically verifies that the dataflow path commutes with the monomorphism inclusions, rejecting any invalid compositions across different domains before generating the blind bytecode [01.7.md, 02.02.md].

## VI. Glossary of Terms for Non-Technical Readers

We define the foundational concepts used in this specification to ensure interdisciplinary readability [1].

### A. Categorical Grammar Terminology
*   **Pregroup Grammar:** A computational algebraic grammar model that assigns adjoint types to words and evaluates grammatical correctness as simple monoidal contractions, eliminating the need for complex syntactic trees.
*   **DisCoCat (Compositional Distributional Semantics):** A category-theoretic framework that fuses grammatical structures with vector spaces, representing the meaning of a sentence as the directed flow of word-meaning through a monoidal graph.
*   **Symmetric Monoidal Category (SMC) Wires:** A programming model representing dataflow as parallel, non-overlapping wires and boxes, allowing concurrent operations to execute without a sequential program counter.

### B. Semantic and Scale Terminology
*   **Qualitative Zoom ($z_{\text{qual}}$):** A type parameter representing the structural abstraction depth or authority tier of a variable (from absolute physical constants to speculative theories), preventing unverified data from corrupting core registers.
*   **Quantitative Zoom ($z_{\text{quant}}$):** A type parameter representing the metric precision of a variable, used to dynamically align scales before register calculations.
*   **Functorial Semantics:** A category-theoretic method of assigning meaning to a system, where the structural relationships between objects are preserved exactly during translation.
