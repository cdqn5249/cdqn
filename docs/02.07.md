## II. Architectural Specifications of the Stage 1 Testbed

We specify the resource-management and parsing architectures of the Stage 1 testbed, demonstrating how the compiler maps abstract invariants directly to physical stack instructions [2, 10].

### A. The $p$-adic Tree-Structured Slab Allocator Layout
Grounded in Bala Subramanyan (2025) and Robert N. M. Watson et al. (2015), we compartmentalize the flat WebAssembly linear memory heap into disjoint, nested **$p$-adic memory balls** to prevent out-of-bounds corruption [36 in 02.06, 19 in 02.03, 65]. 

The `C1_ledger_test.wat` source code partitions the memory array into two distinct, hardware-aligned memory slabs [02.06.md]:
1.  **Slab A (Primary - Offset 16 to 1024):** Allocated exclusively to store the local, memory-isolated data stack of the primary execution thread [02.05.md, 02.06.md].
2.  **Slab B (Secondary - Offset 1024 to 2048):** Allocated exclusively to store the speculative, asynchronous $5$-adic background remainder thunks, preventing cross-core cache invalidations [02.06.md, 02.06a.md].

We propose to represent the memory pointers strictly as **opaque reference handles** (simulated via an index pointer local variable `local $hp i32` mapping to private Wasm Tables) [02.03.md, 02.06.md]. 

The guest program is physically barred from executing pointer arithmetic or forging an address [36, 37 in 02.06]. When executing bare-metal under `cdqnK`, these handles map directly to physical 128-bit CHERI hardware capability registers, enforcing spatial safety at the CPU gate level [36 in 02.06].

### B. The Symmetric Monoidal Parsing Engine Layout
Grounded in Joachim Lambek (2008) and Bob Coecke et al. (2010), the parser reads the sequential input tokens on-the-fly and reduces them using pregroup monoidal contractions [10, 27 in 02.05]. 
*   **The Contraction:** Wires are compiled as Single-Writer Wires [14 in 02.03], and grammatical reductions are computed purely as monoidal contractions (cup-like wires), directly constructing the flat $QnIR_1$ hypergraph dataflow with zero heap allocation [22, 26 in 02.02].
*   **The Destructor:** If an affine variable (`~`) is left unconsumed at a branch termination point, the parser automatically injects an explicit deallocation morphism (the `$discard` operator) to scrub and zero out the memory slab [15 in 03.02, 02.03.md].

## III. The Speculative Specifier and Thread Alignment

We specify how the parsing engine manages the asymmetric speculative coprime thread split ($\mathbb{Q}_{10} \cong \mathbb{Q}_2 \times \mathbb{Q}_5$) concurrently on physical multi-core processors [`CDQN-02.06a`].

### A. The Asymmetric Coprime Thread Split
To prevent the slow, multi-precision 5-adic software remainder thread from blocking the fast, single-cycle 2-adic hardware ALU thread, the compiler splits the dataflow into parallel monoidal wires [`CDQN-02.06a`]:
1.  **The Primary Thread (Core 1):** Executes the 2-adic integer component at maximum hardware frequency, completely unblocked by the 5-adic software emulation [`CDQN-02.06a`].
2.  **The Secondary Thread (Core 2):** Executes the multi-precision rational arithmetic and Schönhage Fast GCD reductions as a low-priority, asynchronous software task in Slab B [02.06.md, `CDQN-02.06a`].

### B. The Speculative Barrier Integration
Grounded in Santiago Arranz Olmos et al. (2025) and Gilles Barthe et al. (2021), we statically inject **speculative barriers** at the dynamic token propagation and thread-join boundaries to prevent Core 1 from speculatively leaking state [64 in 02.04, `CDQN-02.06a`, 66]. 

The primary core is physically blocked from executing speculatively ahead until the asynchronous 5-adic remainder calculation on Core 2 is completed and verified, completely neutralizing Spectre-style cache side-channels [64 in 02.04]. The terminal thread-join and zero-equivalence checks are executed strictly via the branch-free `select` operator, ensuring timing-attack immunity [02.04.md, 02.06a.md].

```wat
(module
  ;; 1. Import WASI Preview 1 system calls (MUST occur first)
  (import "wasi_snapshot_preview1" "proc_exit" (func $proc_exit (param i32)))

  ;; 2. Declare 1 page of memory and export it for WASI compliance
  (memory (export "memory") 1)

  ;; 3. Define a safe, stack-only 2-adic addition function (add2)
  (func $add2 (param $x i64) (param $y i64) (result i64)
    local.get $x
    local.get $y
    i64.add
  )

  ;; 4. Define a safe, stack-only 2-adic multiplication function (mul2)
  (func $mul2 (param $x i64) (param $y i64) (result i64)
    local.get $x
    local.get $y
    i64.mul
  )

  ;; 5. Define a safe, non-Cartesian affine destructor function (discard_i64)
  ;; It has NO result type, meaning it natively consumes and deallocates the parameter.
  (func $discard_i64 (param $wire i64)
    i64.const 0
    local.set $wire  ;; Active register scrubbing to prevent data remanence
  )

  ;; 6. Helper function: Cleanly scrub the stack memory and exit to prevent data remanence on trap
  ;; This physically executes the Guest-Level Graceful Affine Teardown upon guest traps or aborts
  (func $exit_with_scrub (param $exit_code i32) (param $sp i32)
    (local $addr i32)
    (local $i i32)
    
    ;; Loop from index i = 0 up to sp - 1
    i32.const 0
    local.set $i
    
    (loop $scrub_loop
      local.get $i
      local.get $sp
      i32.lt_s
      if
        ;; Calculate memory address in Slab A: 16 + (i * 8)
        local.get $i
        i32.const 8
        i32.mul
        i32.const 16
        i32.add
        local.set $addr
        
        ;; Overwrite un-evaluated memory cell with 0
        local.get $addr
        i64.const 0
        i64.store
        
        ;; Increment i
        local.get $i
        i32.const 1
        i32.add
        local.set $i
        
        br $scrub_loop
      end
    )
    
    ;; Invoke WASI proc_exit with the provided exit code
    local.get $exit_code
    call $proc_exit
  )

  ;; 7. Helper function: Push an i64 onto Slab A data stack (starts at offset 16)
  ;; Spatial bounds check: prevent stack overflow (max stack size 125 elements, up to offset 1016)
  (func $push (param $val i64) (param $sp i32) (result i32)
    local.get $sp
    i32.const 125
    i32.ge_s
    if
      ;; Native trap bypass prevention: scrub memory before trapping
      i32.const 1       ;; exit code 1
      local.get $sp     ;; current stack pointer
      call $exit_with_scrub
      unreachable       ;; Satisfy Wasm validator: this path is dead
    end

    ;; Calculate target memory offset in Slab A: 16 + (sp * 8) and store the value
    local.get $sp
    i32.const 8
    i32.mul
    i32.const 16
    i32.add
    local.get $val
    i64.store

    ;; Return the incremented stack pointer (sp + 1)
    local.get $sp
    i32.const 1
    i32.add
  )

  ;; 8. Helper function: Pop an i64 value from Slab A data stack
  ;; Spatial bounds check: prevent stack underflow
  (func $pop_val (param $sp i32) (result i64)
    local.get $sp
    i32.const 0
    i32.le_s
    if
      ;; Native trap bypass prevention: scrub memory before trapping
      i32.const 1       ;; exit code 1
      local.get $sp     ;; current stack pointer
      call $exit_with_scrub
      unreachable       ;; Satisfy Wasm validator: this path is dead
    end

    ;; Calculate source memory offset of the top element: 16 + ((sp - 1) * 8)
    local.get $sp
    i32.const 1
    i32.sub
    i32.const 8
    i32.mul
    i32.const 16
    i32.add
    
    ;; Load and return the value
    i64.load
  )

  ;; 9. Simulated 5-adic Emulation Thread (runs asynchronously, represented as low-priority Slab B write)
  ;; This simulates the asymmetric split by storing the 5-adic remainder in Slab B (starts at offset 1024)
  (func $write_5adic_cofactor (param $cofactor i64)
    ;; Store the 5-adic cofactor at offset 1024 in Slab B
    i32.const 1024
    local.get $cofactor
    i64.store
  )

  ;; 10. The terminal, branch-free constant-time Soft Halt operator (shalt)
  (func $shalt (param $numerator i64)
    (local $status i32)
    
    ;; Flatten control-flow via stack-based select:
    ;; If numerator is 0, exit status is 0 (success).
    ;; If numerator is non-zero, exit status is 1 (failure).
    i32.const 0       ;; val1 (value if cond is non-zero, meaning numerator == 0 -> success status 0)
    i32.const 1       ;; val2 (value if cond is zero, meaning numerator != 0 -> failure status 1)
    local.get $numerator
    i64.eqz           ;; cond (pushes 1 if numerator == 0, else 0)
    select            ;; Pops condition, returns 0 if true, 1 if false (CMOVcc mapping)
    local.set $status
    
    local.get $status
    call $proc_exit
  )

  ;; Main entry point of the Stage 1 verification suite
  (func (export "_start")
    (local $sp i32)       ;; Stack pointer for Slab A
    (local $sum i64)
    (local $product i64)
    (local $val1 i64)     ;; Temporary registers for addition operations
    (local $val2 i64)

    i32.const 0
    local.set $sp

    ;; --- TEST 1: Allocate and Push exact ledger inputs onto Slab A ---
    i64.const 9           ;; balance_a
    local.get $sp
    call $push
    local.set $sp

    i64.const 0           ;; balance_b
    local.get $sp
    call $push
    local.set $sp

    ;; --- TEST 2: Perform 2-adic Addition ---
    local.get $sp
    call $pop_val
    local.set $val1        ;; val1 = 0
    
    local.get $sp
    i32.const 1
    i32.sub
    call $pop_val
    local.set $val2        ;; val2 = 9

    local.get $val2
    local.get $val1
    call $add2
    local.set $sum        ;; sum = 9 + 0 = 9

    ;; Adjust stack pointer after consuming two elements
    local.get $sp
    i32.const 2
    i32.sub
    local.set $sp

    ;; --- TEST 3: Simulate Asymmetric Split ---
    ;; Write the 5-adic software remainder (0) to Slab B (Offset 1024)
    i64.const 0
    call $write_5adic_cofactor

    ;; --- TEST 4: Perform 2-adic Multiplication ---
    ;; Multiply sum (9) by transfer_amount (0)
    local.get $sum
    i64.const 0           ;; transfer_amount
    call $mul2
    local.set $product    ;; product = 9 * 0 = 0

    ;; --- TEST 5: Execute shalt to resolve zero-equivalence ---
    local.get $product
    call $shalt
  )
)
```

## VI. Glossary of Terms for Non-Technical Readers

We define the foundational concepts used in this specification to ensure interdisciplinary readability [1].

### A. Substructural and Allocator Terminology
*   **$p$-adic Tree-Structured Slab Allocator:** A memory management model that partitions the flat WebAssembly linear memory heap into disjoint, non-overlapping $p$-adic memory balls, providing spatial capability bounds checking natively.
*   **Single-Writer Wire:** A security constraint where every data channel has exactly one producer and one consumer, preventing data-race hazards across parallel processor cores.
*   **Host-Level Affine Teardown:** A security recovery protocol where the host operating system or kernel catches guest-level traps, walks the capability tables, and manually zeroes out speculative memory segments before recycling the sandbox to prevent data-remanence leaks.

### B. Parsing and Threading Terminology
*   **Symmetric Monoidal Parsing Engine:** A computational parsing engine that translates high-level natural language tokens directly into strict category-theoretic string diagrams using linear-time pregroup reductions.
*   **Asymmetric Speculative Coprime Thread Split:** A multicore task scheduling model that decouples the fast, single-cycle 2-adic hardware ALU execution from the slow, multi-precision 5-adic background software remainder thread, maximizing processing speed.
*   **Speculative Barrier:** A hardware-level CPU instruction (such as `CSDB` or `FENCE`) that physically blocks speculative execution past a synchronization point, neutralizing Spectre-style cache side-channels.
