I. The Inquiring Spark

This research project originates from a single, fundamental inquiry:

"Can a computing number system abstract anything?"

If Kurt Gödel demonstrated that logical statements can be mapped to unique natural numbers, and if modern Artificial Intelligence demonstrates that multi-dimensional vectors can represent human reasoning, semantics, and intent, then mathematically the answer is affirmative: A computing number can indeed abstract any system.

However, when we translate this mathematical premise into software systems, we encounter a fundamental divergence in design philosophy. Traditional computer science has historically defined abstraction as hiding complexity—encapsulating intricate operations beneath opaque boundaries.

In an era dominated by opaque AI latent spaces, autonomous multi-agent interactions, and distributed zero-trust networks, hiding complexity yields inscrutable black boxes, non-deterministic behaviors, and unverified data flows.

Therefore, our initial question forces a primary deduction: If a computing number system is to abstract anything reliably, it must redefine abstraction from hiding complexity to structuring complexity.

From this deduction emerges the concept of a Quang Number (Qn). Derived from the Vietnamese word Quang—signifying luminescence, brightness, and clarity—a Quang Number is a computing number designed not to conceal state, but to illuminate it by structuring complexity into observable, verifiable mathematical relationships.

II. Identified CS Friction Points

Our objective is not to criticize established computer science paradigms, which have successfully powered decades of digital transformation. Rather, we identify specific architectural friction points that naturally arise when existing paradigms are stretched into autonomous, highly distributed, agentic environments.

The SMEMP Manifesto is designed as a logical set of constraints specifically crafted to address these friction points:

1. The Opacity & Sandboxing Friction Addressable by [S]

Treating data as "passive content" while relying on external runtime sandboxing creates severe vulnerabilities. As autonomous agents dynamically interpret data, configuration files, and templates, passive data becomes an active execution path (e.g., indirect prompt injections, malicious dataset payloads).

2. The Semantic Ambiguity Friction Addressable by [M]

Modern vector spaces in AI provide high expressiveness but often lack strict, deterministic bounds or human/machine-verifiable geometric metrics, introducing non-deterministic drift and unpredictable evaluation paths.

3. The Trust-vs-Bloat Friction Addressable by [E]

Decentralized state verification (such as traditional blockchain consensus) frequently trades performance for trust, requiring global ledger replication that causes exponential storage, energy, and latency bloat incompatible with consumer devices.

4. The Latency & Allocation Friction Addressable by [Mod]

Dynamic object-oriented heap allocation (`malloc`/`free`) and managed garbage collection cause unpredictable micro-pauses and memory fragmentation, degrading real-time responsiveness on low-power consumer hardware.

5. The Platform Lock-In Friction Addressable by [P]

Coupling high-level software models directly to heavy language runtimes (e.g., JVM, V8) or volatile hardware APIs limits execution fluidness across heterogeneous edge devices, NPUs, and web views.

III. The Invariants (S × M) — Deduce the Birth Laws

To resolve the Opacity and Ambiguity friction points without sacrificing system integrity, we deduce that a Quang Number cannot exist as a raw, floating value. It must be instantiated alongside two mandatory, non-negotiable invariants:

Birth of Qn ⟺ ( Security [S] ≥ Cryptographic_Threshold ) ∧ ( Metric [M] ∈ Finite_Metric_Space )

[S] Security — Intrinsic Provenance & Identity

A Qn carries its own cryptographic identity, state immutability proof, and execution boundary natively. Security is not an external sandbox wrapper added later; it is an intrinsic invariant. Unauthenticated or untrusted data cannot be "born" as a valid Qn.

[M] Metric — Measurable & Deterministic Space

To structure complexity, state must be quantifiable. A Qn resides within a computable, deterministic metric space $(X, d)$ over discrete sets. Distance, proximity, and transformation between two states ($Q_a, Q_b$) must be calculable in finite time without ambiguity.

IV. The Agile Trinity (E, Mod, P) & Best of Choices (BoC)

While the Invariant Couple ($S \times M$) governs existence, the Agile Trinity governs efficiency and consumer adaptation. Unlike the invariants, the three agile criteria are independent optimization vectors tuned by the Best of Choices (BoC) dynamic engine:

  • [E] Efficiency (Recursive Accumulator Folding): To eliminate the Trust-vs-Bloat friction, $E$ enforces constant $O(1)$ space verification. Past state chains are algebraically folded into single accumulator Qns, eliminating the need to store massive transaction histories on consumer devices.
  • [Mod] Modularity (Arena Value-Semantics): To eliminate memory fragmentation and garbage collection pauses, Qn primitives use flat value-semantics allocated within pre-managed memory arenas. Composition and decomposition are bitwise operations.
  • [P] Portability (Hardware Neutrality via QnIR): To eliminate platform lock-in, $P$ decouples the high-level intent from the physical hardware. Source code in `QnLang` compiles to `QnIR` (Intermediate Representation), which emits lightweight targets for WASM/WAT, bare-metal RISC-V, ARM, or x86.

The Best of Choices (BoC) Engine: In noisy, power-constrained, real-world environments, BoC acts as the dynamic policy layer that optimizes $E, Mod, P$ according to current runtime context (e.g., thermal limits, battery levels, network noise) to preserve maximum CDQN network stability.

V. System Constraints: Finitism & Halting

From the requirement of deterministic, real-time evaluation, we derive three fundamental system laws:

1. Finitism & Bounded Model Checking

All Qn and CDQN operations take place strictly over bounded, discrete, finite sets. Verification does not depend on asymptotic infinite mathematical induction, but on exhaustive, provable bounded model checking and SMT solvers over finite metric lattices.

2. Total Functionality (Turing Incompleteness)

To guarantee the Metric invariant ($M$), execution cannot suffer from the undecidability of the Halting Problem. All `QnIR` operations are total (Turing Incomplete) with provable upper execution bounds ($O(1)$ or $O(N)$), ensuring deterministic termination prior to execution.

3. Fractal Homoiconicity

Because $(S \times M)$ is the universal condition for existence, every component in the stack—a primitive data byte, an operation, an AI agent state, or a network node—is fundamentally a Quang Number. There are no "unprotected raw bytes" floating outside the Qn boundary.

VI. The Boundaries of Abstraction

To remain anchored in physical reality, the SMEMP Manifesto explicitly defines where abstraction begins and ends:

CEILING BOUNDARY (Semantic Clarity / Base10):
Human & AI Agent Interface. Structured, observable relationships, visual rendering (HTML/Dual-channel), and human-readable semantic intent.
MIDDLE LAYER (The Qn / CDQN Universe):
Governed by SMEMP Laws, formally verified by QnLang & QnIR, executing fractal Qn state chains.
FLOOR BOUNDARY (Silicon Ground Truth / Base2):
Bare-metal SoC registers, bitwise execution, and physical voltage states. WHERE ABSTRACTION ENDS.
