1.  **The Heap Dependency:** They require allocating a dynamic heap to parse source text, construct an Abstract Syntax Tree (AST), and traverse the execution nodes [5]. 2.  **The Performance Penalty:** Standard virtual machines rely on heavy runtime garbage collectors and JIT compiler threads that run continuously, consuming CPU clock cycles, generating heat, and triggering physical thermal throttling on resource-constrained hardware (Option C) [24, 25].

### A. The S-Expression Tokenizer
The $QnIR_0$ bytecode represents compiled dataflow string diagrams as linear, sequential S-expressions [11]. The interpreter imports the serialized bytecode stream from the WASI standard input (`fd_read`) as a contiguous flat array of bytes [9]. 

The tokenizer is written as a simple, stateless state-machine in $Wat_{\text{safe}}$ [1]. It reads the byte stream sequentially, identifying the start parenthesization `(`, the end parenthesization `)`, and the exact instruction keywords (`add2`, `mul2`, `sym`, `discard_i64`, `shalt`) [2, 11]:
*   **The Tokenizer Loop:** Since the input characters are read and matched on-the-fly, the tokenizer requires **zero heap memory allocation** [1]. 
*   It passes the matched instruction tokens directly to the parsing stack [1].

### B. The Pushdown Automaton Parser
The parser is implemented as a strict **Pushdown Automaton (PDA)** executing natively on Wasm’s typed operand stack [1].
*   **The Parsing Rule:** Following the classic single-pass translation schemes of compiler theory, when the parser encounters an operand token (such as a 64-bit $2$-adic numeric constant `i64_z`), it pushes the value directly onto the stack [1, 24].
*   **The Execution Rule:** When the parser encounters an instruction token (such as the addition morphism `add2`), it instantly pops the required operand values from the stack, executes the respective function natively, and leaves the computed result on the stack [1, 2].

We hypothesize that this single-pass, stack-bound parser executes with zero memory allocation or temporary register spilling, maintaining native-speed execution without requiring complex runtime control-flow engines [1, 5, 23]. `[Empirical Horizon]`

### A. The C0_interpreter.wat Source Code
WebAssembly's validation algorithm strictly enforces that the operand stack height must be completely invariant and statically determinable at all branch targets (`br` / loop boundaries) [1, 16]. Directly using Wasm's operand stack to hold our dynamic data stream (where `digit` pushes +1 and `add2` pops -1 inside a loop) is forbidden, and will trigger a fatal `type mismatch` compilation error in the host validator [1, 16]. 

To satisfy the Wasm validator while maintaining our "zero dynamic heap allocation" SEMP constraint, **we propose to map the PDA's dynamic data stack to a strictly bounded array in our allocated linear memory [1, 16].** `[Empirical Horizon]` We use a local variable `(local $sp i32)` as an explicit **stack pointer** to push and pop `i64` values via `i64.store` and `i64.load` [1, 16]. To satisfy the WASI Preview 1 host requirements, we use `local.set $unused` [1] to pop and consume the `fd_read` return status, avoiding the un-algebraic `drop` instruction [1].

Furthermore, to enforce the **Guest-Level Graceful Affine Teardown** upon guest-level traps or aborts, we construct the helper function **`$exit_with_scrub`** [3]. This function walks the data stack from offset 16 up to `$sp` and overwrites every active memory cell with `0` (active register/memory scrubbing) before calling `proc_exit 1`, preventing data remanence exploits [15, 17]. To prevent native Wasm traps (`unreachable`) from bypassing this guest-level scrubber, all stack overflow and underflow bounds checks inside `$push` and `$pop_val` are updated to call `$exit_with_scrub` immediately before executing `unreachable` [17, 16]. True "Host-Level" capability-based scrubbing (where the bare-metal kernel walks an `externref` table upon hardware-level traps) is deferred to Stage 1 (`CDQN-02.06`), and Stage 0 relies on guest-side graceful scrubbing and host OS page reclamation for native traps [17, 19, 15].

(module

  (import "wasi_snapshot_preview1" "fd_read" (func $fd_read (param i32 i32 i32 i32) (result i32)))
  (import "wasi_snapshot_preview1" "proc_exit" (func $proc_exit (param i32)))

  ;; 2. Declare 1 page of memory and export it for WASI compliance
  (memory (export "memory") 1)

  ;; 3. Helper function: Read a single character byte from WASI stdin (fd 0)
  (func $read_char (result i32)
    (local $unused i32)
    
    ;; Initialize __wasi_ciovec_t structure at offset 0
    i32.const 0
    i32.const 8      ;; iov_base points to offset 8
    i32.store
    
    i32.const 4
    i32.const 1      ;; iov_len set to 1 byte
    i32.store

    ;; Call fd_read (fd=0, iovs=0, iovs_len=1, nread=12)
    i32.const 0      ;; fd (stdin)
    i32.const 0      ;; iovs pointer
    i32.const 1      ;; iovs_len
    i32.const 12     ;; nread pointer
    call $fd_read
    local.set $unused ;; Pop and consume the syscall return status without using "drop"

    ;; Check if we successfully read exactly 1 byte
    i32.const 12
    i32.load
    i32.const 1
    i32.ne
    if
      ;; Return -1 on EOF (End of File) or Read Error
      i32.const -1
      return
    end

    ;; Load and return the 1 byte of data from offset 8
    i32.const 8
    i32.load8_u
  )

  ;; 4. Safe, stack-only 2-adic addition function (add2)
  (func $add2 (param $x i64) (param $y i64) (result i64)
    local.get $x
    local.get $y
    i64.add
  )

  ;; 5. Safe, stack-only 2-adic multiplication function (mul2)
  (func $mul2 (param $x i64) (param $y i64) (result i64)
    local.get $x
    local.get $y
    i64.mul
  )

  ;; 6. Safe, stack-only wire swap function (representing sym)
  (func $sym (param $a i64) (param $b i64) (result i64 i64)
    local.get $b
    local.get $a
  )

  ;; 7. Safe, non-Cartesian affine destructor function (discard_i64)
  ;; It has NO result type, meaning it natively consumes and deallocates the parameter.
  ;; Note: In Wasm, the local parameter is copied into the local frame. Overwriting it
  ;; serves as a logical type-sink and memory-scrubbing marker before the frame collapses.
  (func $discard_i64 (param $wire i64)
    i64.const 0
    local.set $wire  ;; Active register scrubbing to prevent data remanence
  )

  ;; 8. Helper function: Cleanly scrub the stack memory and exit to prevent data remanence on trap
  ;; This physically executes the Guest-Level Graceful Affine Teardown upon guest traps or aborts
  (func $exit_with_scrub (param $exit_code i32) (param $sp i32)
    (local $addr i32)
    (local $i i32)
    
    ;; Loop from index i = 0 up to sp - 1
    i32.const 0
    local.set $i
    
    (loop $scrub_loop
      local.get $i
      local.get $sp
      i32.lt_s
      if
        ;; Calculate memory address: 16 + (i * 8)
        local.get $i
        i32.const 8
        i32.mul
        i32.const 16
        i32.add
        local.set $addr
        
        ;; Overwrite un-evaluated memory cell with 0
        local.get $addr
        i64.const 0
        i64.store
        
        ;; Increment i
        local.get $i
        i32.const 1
        i32.add
        local.set $i
        
        br $scrub_loop
      end
    )
    
    ;; Invoke WASI proc_exit with the provided exit code
    local.get $exit_code
    call $proc_exit
  )

  ;; 9. Helper function: Push an i64 onto our memory-isolated data stack (starts at offset 16)
  (func $push (param $val i64) (param $sp i32) (result i32)
    ;; Spatial bounds check: prevent stack overflow (max stack size 8190 elements)
    local.get $sp
    i32.const 8190
    i32.ge_s
    if
      ;; Native trap bypass prevention: scrub memory before trapping
      i32.const 1       ;; exit code 1
      local.get $sp     ;; current stack pointer
      call $exit_with_scrub
      unreachable       ;; Satisfy Wasm validator: this path is dead
    end

    ;; Calculate target memory offset: 16 + (sp * 8) and store the value
    local.get $sp
    i32.const 8
    i32.mul
    i32.const 16
    i32.add
    local.get $val
    i64.store

    ;; Return the incremented stack pointer (sp + 1)
    local.get $sp
    i32.const 1
    i32.add
  )

  ;; 10. Helper function: Pop an i64 value from our memory-isolated data stack
  (func $pop_val (param $sp i32) (result i64)
    ;; Spatial bounds check: prevent stack underflow
    local.get $sp
    i32.const 0
    i32.le_s
    if
      ;; Native trap bypass prevention: scrub memory before trapping
      i32.const 1       ;; exit code 1
      local.get $sp     ;; current stack pointer
      call $exit_with_scrub
      unreachable       ;; Satisfy Wasm validator: this path is dead
    end

    ;; Calculate source memory offset of the top element: 16 + ((sp - 1) * 8)
    local.get $sp
    i32.const 1
    i32.sub
    i32.const 8
    i32.mul
    i32.const 16
    i32.add
    
    ;; Load and return the value
    i64.load
  )

  ;; 11. The terminal Soft Halt operator (shalt)
  (func $shalt (param $numerator i64)
    local.get $numerator
    i64.eqz
    if
      i32.const 0   ;; Push success status 0 onto stack
      call $proc_exit
    else
      i32.const 1   ;; Push failure status 1 onto stack
      call $proc_exit
    end
  )

  ;; Main entry point of the Pushdown Automaton interpreter
  (func (export "_start")
    (local $char i32)
    (local $sp i32)       ;; Our explicit data stack pointer (initialized to 0)
    (local $val1 i64)     ;; Temporary registers for operations
    (local $val2 i64)

    i32.const 0
    local.set $sp

    (loop $parse_loop
      ;; Read next character from the WASI stream
      call $read_char
      local.set $char

      ;; Check for EOF (-1)
      local.get $char
      i32.const -1
      i32.eq
      if
        ;; If we hit EOF before shalt, scrub memory and exit with failure status 1
        i32.const 1
        local.get $sp
        call $exit_with_scrub
      end

      ;; Skip whitespace: Space (32), LF (10), CR (13)
      local.get $char
      i32.const 32
      i32.eq
      local.get $char
      i32.const 10
      i32.eq
      i32.or
      local.get $char
      i32.const 13
      i32.eq
      i32.or
      if
        br $parse_loop
      end

      ;; --- THE PARSER ALGEBRAIC MAPPINGS ---

      ;; 1. Check for Addition Operator '+' (ASCII 43)
      local.get $char
      i32.const 43
      i32.eq
      if
        ;; Pop top two elements, add them, and push result back
        local.get $sp
        call $pop_val
        local.set $val1
        
        local.get $sp
        i32.const 1
        i32.sub
        call $pop_val
        local.set $val2

        local.get $val2
        local.get $val1
        call $add2
        
        local.get $sp
        i32.const 2
        i32.sub
        call $push
        local.set $sp
        br $parse_loop
      end

      ;; 2. Check for Multiplication Operator '*' (ASCII 42)
      local.get $char
      i32.const 42
      i32.eq
      if
        ;; Pop top two elements, multiply them, and push result back
        local.get $sp
        call $pop_val
        local.set $val1
        
        local.get $sp
        i32.const 1
        i32.sub
        call $pop_val
        local.set $val2

        local.get $val2
        local.get $val1
        call $mul2
        
        local.get $sp
        i32.const 2
        i32.sub
        call $push
        local.set $sp
        br $parse_loop
      end

      ;; 3. Check for Symmetry Wire Swap 's' (ASCII 115)
      local.get $char
      i32.const 115
      i32.eq
      if
        ;; Pop top two, and write them back swapped in memory
        local.get $sp
        call $pop_val
        local.set $val1

        i32.const 1
        i32.sub
        call $pop_val
        local.set $val2

        ;; Write back swapped: val1 to index sp-2, val2 to index sp-1
        local.get $sp
        i32.const 2
        i32.sub
        i32.const 8
        i32.mul
        i32.const 16
        i32.add
        local.get $val1
        i64.store

        local.get $sp
        i32.const 1
        i32.sub
        i32.const 8

        i32.const 16

        i32.const 1
        i32.sub
        local.set $sp

## VII. Glossary of Terms for Non-Technical Readers

*   **Symmetric Monoidal Category (SMC):** A mathematical structure where calculations are represented as diagrams of parallel wires (representing data states) and boxes (representing operations), allowing parallel and sequential steps to be compiled without a traditional program counter.
*   **Pushdown Automaton (Parser):** A mathematical parser model that uses a strictly typed stack to parse and execute program instructions on-the-fly, completely bypassing the need for a complex, heap-allocated Abstract Syntax Tree (AST).
*   **Symmetric Monoidal Stack Invariant:** A compile-time constraint enforcing that every stack register representing a linear resource must be read exactly once, preventing memory leaks and use-after-free exploits.
*   **Algebraic Effect System:** A type-safety mechanism in modern compilers that treats system actions (like I/O, file writing, or GPU dispatches) as formal algebraic operations. The compiler statically guarantees that no library can execute sneaky or unrequested background processes.
*   **Regulation Sheaf ($\mathcal{R}_{\text{reg}}$):** The type-checking constraint in QnLang that mathematically monitors and enforces software license compliance at compile-time, ensuring copyleft protection.
