### 1.3 Complexity Claims (§5.2)

**Claim:** Verification is O(1) in sequence length N and O(D) in dimensionality D via Sparse Manifold Projection.

**Analysis:**

For **1D word verification** (single dimension):
- Local neighborhood check with radius R=3
- Neighborhood size: 2R+1 = 7 points
- Cost per position: O(7) = O(1) with respect to N ✓

**For 1024D stacked manifold:**
- If checking k dimensions per pulse where k << D
- Cost: O(k × constant) = O(k)
- If k scales as O(D): total cost O(D) ✓
- If k is constant: cost O(1) with respect to D also ✓

**Honesty assessment:** The claim is **mathematically conservative** (stating O(D) upper bound). The mechanism (sparse projection onto k orthogonal planes) is reasonable but lacks formal analysis of:
- Selection strategy for k planes
- False-negative rate (missed phasons due to sparse checking)
- Optimal k value for D=1024

**Status:** Acceptable bound with appropriate [†2] marking.

## 3. PHYSICAL CORRECTNESS

### 3.1 Energy Consumption Claim (Appendix A)

**Claim:** Node Beta recorded average of **12 nJ** per sovereign act (4 heartbeats).

**Plausibility check:**

**Modern CPU operation energy costs (45nm process, ~2GHz):**
- 32-bit integer ADD: ~0.1 pJ
- 32-bit XOR: ~0.1 pJ
- L1 cache access: ~0.5 pJ
- Register read/write: ~0.05 pJ each

**StM-Loop-T operations per heartbeat (estimated):**
1. READ_CLOCK (TSC): ~50 cycles ≈ 5 pJ
2. EXECUTE (XOR R1, 1): ~1 cycle ≈ 0.1 pJ
3. READ_CLOCK (TSC): ~50 cycles ≈ 5 pJ
4. DELTA calculation: ~5 cycles ≈ 0.5 pJ
5. σ application: ~10 cycles ≈ 1 pJ
6. Memory write: ~20 cycles ≈ 2 pJ

**Per heartbeat:** ~13.6 pJ  
**4 heartbeats:** ~54.4 pJ = **0.0544 nJ**

**Discrepancy:** Document claims 12 nJ, calculation suggests ~0.05 nJ (difference of ~220×)

**Possible explanations:**
1. Document includes overhead (OS scheduling, memory controller, etc.)
2. Uses older/larger process node estimates
3. Includes full subsystem activity (not just CPU core)
4. Conservative upper bound intentionally stated

**Status:** Value seems high by 2-3 orders of magnitude for raw operations, but may be defensible if including system-level overhead. Should clarify methodology.
