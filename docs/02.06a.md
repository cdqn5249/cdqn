## I. Introduction and the Asymmetric Latency Gap

In Chapter 1.4 and 1.13, we formulated the **Coprime Base Decomposition** to prevent representation loss and floating-point rounding drift, proving that the ring of $10$-adic integers decomposes exactly into the product of the $2$-adic and $5$-adic integers [4, 5]. At the physical register layer, this allows a decimal variable to be split into a native binary register and an exact, symbolic fractional remainder carried as a cofactor [4].

However, executing these split variables concurrently introduces a severe microarchitectural friction:

*   **The Latency Gap:** The 2-adic integer component is processed natively by the physical CPU ALU in exactly $1$ hardware clock cycle [1]. Conversely, the 5-adic symbolic cofactor requires software-emulated rational arithmetic (multi-precision, fraction simplification via Schönhage Fast GCD), taking hundreds of clock cycles [10, 11].
*   **The Serialization Failure:** If the native hardware thread must block and wait for the slow 5-adic software emulation thread to complete on every single arithmetic step, the system loses all native hardware acceleration [10]. The processing throughput degrades to the speed of pure software emulation, violating our strict **Efficiency** constraint [10].
*   **The Required Solution:** We must decouple the execution pipelines [2]. We propose to let the fast 2-adic hardware thread run speculatively ahead on the primary execution core, while the slow 5-adic software remainder is calculated asynchronously on an isolated, low-priority background thread on a secondary core [2, 12, 19].

### A. Bypassing the Dijkstra Semantic Gap
The semantic gap between abstract category theory and physical machine code is a documented software engineering friction [35]. In traditional compilers, direct lowering to physical registers forces the compilation pipeline to handle low-level optimization complexities—such as register-allocation and platform-specific memory boundaries—early in the design cycle [10]. By targeting a virtual, stack-based Instruction Set Architecture (vISA), we isolate the core compiler logic, allowing the system to run on a highly structured, mathematically tractable, and target-independent dataflow model before physical code generation [10, 14].

## II. The Asymmetric Speculative Thread Split

To compile the coprime register split into $QnIR_1$ with zero-lock overhead, we model the thread allocation as a parallel composition of monoidal wires [2].

Let $A = [(\text{i64}, \; z)]$ and $B = [(\text{i64}, \; z)]$ be two scaled input wires. We compile the parallel execution of the split registers as a monoidal tensor product morphism [2]:

$$f_{\text{split}} : A \to A_2 \otimes A_5$$

Where:
*   $A_2 = [(\text{i64}, \; z)_2]$ represents the native $2$-adic hardware wire [4].
*   $A_5 = [(\text{i64}, \; z)_5]$ represents the symbolic $5$-adic software cofactor wire [4].

```
                     THE ASYMMETRIC MONOIDAL SPLIT
    Wire A ───► [ f_split ] ───┬───► Wire A_2 (2-adic) ───► Primary Core (ALU Speed)
                               │
                               └───► Wire A_5 (5-adic) ───► Secondary Core (Software Thread)
```

The compiler ($QnLang_1$) lowers this split to the virtual ISA (`Wasm/WASI` under Option C, or `cdqnK` under Option A) by emitting a parallel **Asymmetric Thread-Spawn instruction** [1, 2]:
1.  **The Primary Thread:** The native 2-adic wire $A_2$ is mapped directly to standard CPU registers on Core 1 [1]. It executes raw integer arithmetic (`i64.add` or `i64.mul`) at maximum hardware frequency, completely unblocked by the 5-adic software emulation [1, 10].
2.  **The Secondary Thread:** The 5-adic cofactor wire $A_5$ is passed as an opaque capability-handle (`externref`) to a background thread spawned on Core 2 [1, 8]. Core 2 executes the multi-precision rational arithmetic and the Schönhage Fast GCD reductions as a low-priority, asynchronous software task [11, 12].

To optimize the thread allocation, the scheduler implements **Asymmetric Task Deployment** [15]. By analyzing the computational complexity of the tasks, the scheduler maps the lightweight, regular 2-adic hardware execution to the high-frequency primary core, and delegates the irregular, multi-precision 5-adic software emulation to the low-frequency background core, preventing register contention [12, 15].

## III. Speculative Synchronization and Saga-Based Realignment

Because the 2-adic hardware thread executes speculatively ahead of the 5-adic background thread, the system must mathematically realign and reconcile the cofactors at designated boundaries [2].

### A. The Realignment Equation
Reconstruction of the exact decimal value $x \in \mathbb{Z}/10^z\mathbb{Z}$ from the split 2-adic register value $x_2 \in \mathbb{Z}/2^z\mathbb{Z}$ and the 5-adic cofactor $x_5 \in \mathbb{Z}/5^z\mathbb{Z}$ at the boundary is computed using the Chinese Remainder Theorem (CRT) isomorphism [2, 4]:

$$\mathbb{Z} / 10^z \mathbb{Z} \cong \mathbb{Z} / 2^z \mathbb{Z} \times \mathbb{Z} / 5^z \mathbb{Z}$$

The reconstruction is given by:

$$x \equiv x_2 \cdot e_2 + x_5 \cdot e_5 \pmod{10^z}$$

Where $e_2, e_5 \in \mathbb{Z}/10^z\mathbb{Z}$ are the modular orthogonal idempotents of the decomposition, satisfying:

$$e_2 \equiv 1 \pmod{2^z}, \quad e_2 \equiv 0 \pmod{5^z}$$

$$e_5 \equiv 0 \pmod{2^z}, \quad e_5 \equiv 1 \pmod{5^z}$$

These idempotents are calculated statically at Zoom scale $z$ using the Bezout coefficients of the coprimality equation $a \cdot 2^z + b \cdot 5^z = 1$, where $e_2 = b \cdot 5^z$ and $e_5 = a \cdot 2^z$.

The prefix-based path routing behaves as a deterministic trie structure [16]. Unlike legacy hashmaps, there are **zero hash collisions**, and the worst-case lookup time is strictly and mathematically bounded by the finite Zoom scale depth: **$O(z)$ [16, 17]**. This deterministic search bound satisfies the hard real-time and SMR execution constraints of the operating system [13].

### B. The Terminal Join Check
This realignment equation is executed strictly at the **boundary-verification gates** (such as serializing data to disk via `wasi_io` or executing the Soft Halt check `shalt`) [14, 20, 21].
*   When the high-level `shalt` node is reached, the speculative 2-adic thread blocks and yields processing cycles [20].
*   It waits for the 5-adic background thread to complete its queue. 
*   Once the background thread resolves, the kernel joins the threads, evaluates the realignment equation, and verifies the zero-equivalence of the remainder ($a = 0$) in $O(1)$ constant time [20].

### C. The Saga-Based Rollback Engine
If the 5-adic background thread calculates that the speculative 2-adic execution has violated a mathematical boundary (such as a division-by-zero, underflow, or an un-glued local state $H^1 \neq 0$), the kernel must execute an immediate rollback [21, 36]:
1.  **The Abort:** The `cdqnK` scheduler immediately cancels the speculative pipeline, discarding any pending thread-join operations [22].
2.  **The Compensating Transaction (Saga):** To safely reverse the speculative local memory frames without leaving stale data or space leaks, the kernel walks the local stack and executes the **affine destructors ($!$)** [23, 36].
3.  **The Register Scrubbing:** It calls our `$discard_i64` void-returning function to actively scrub the speculative registers with `i64.const 0`, preventing memory-remanence or speculative side-channel leaks [1, 23].

This transactional recovery ensures that unverified speculative states are mathematically blocked from ever leaking to persistent disk [21].

## IV. Thread-Join Latency and Contention Management

To execute this asymmetric parallel split without generating performance-degrading cache invalidations or register contention on multi-core SoCs, the allocator implements lock-free contention management [19].

### A. Thread-Local Cache Isolation
If the 2-adic and 5-adic threads constantly read and write to the same shared memory locations, the MESI cache-coherence protocol will trigger continuous cross-core invalidation broadcasts [24]. This generates "cache-coherence storms," stalling the memory bus and degrading processor speed [24].

To bypass this bottleneck, the allocator (`02.06`) enforces strict **Thread-Local Stalk Isolation** [24].
*   The speculative 2-adic register states are mutated exclusively within the L1 cache-line of Core 1 [24].
*   The 5-adic remainder thunk is mutated exclusively within the L1 cache-line of Core 2 [24].
*   Because the threads operate on disjoint, memory-isolated $p$-adic balls, the MESI protocol never triggers cross-core broadcasts, preserving the single-cycle execution speed on the critical path [24].

### B. Lock-Free Contention Management
The thread-join boundary is managed using a **lock-free, single-writer contention management algorithm** [19, 26].
*   The background thread writes its completed remainder $x_5$ to a single-writer, bounded memory slot [22].
*   The primary thread performs a lock-free, atomic memory read to join the state. 
*   Because we completely avoid heavy runtime lock managers (such as semaphores or mutexes), the thread-join latency is mathematically minimized, and the risk of circular deadlocks is eliminated by construction [22, 26].

## VI. Glossary of Terms for Non-Technical Readers

*   **Coprime Base Decomposition:** The algebraic process of breaking down a composite number base (like base 10) into its prime, non-overlapping components (like base 2 and base 5) to allow exact, parallel calculations on computer registers.
*   **Asymmetric Multicore Processor (AMP):** A CPU architecture that combines different types of cores (such as high-speed primary cores and low-power background cores) on a single chip, optimizing the performance-to-energy ratio.
*   **Symmetric Monoidal Category Wires:** A category-theoretic programming model where calculations are represented as diagrams of parallel, non-overlapping wires, allowing parallel instructions to execute without thread locks.
*   **Saga (Compensating Transaction):** A transactional design pattern used to manage distributed recovery. Instead of using heavy database locks, it executes a sequence of local transactions and runs compensating "destructor" actions to reverse states if a failure occurs.
*   **MESI Cache-Coherence Protocol:** A hardware-level protocol used in multi-core processors to keep the local CPU caches synchronized. If multiple cores write to the same memory address, it triggers cache invalidations, slowing the system.
