## II. Ostrowski's Theorem and the $p$-adic Absolute Value

To prevent rounding errors from leaking across scale boundaries, we must define an alternative metric space over our truncated quotients. The mathematical justification for this selection is established by **Ostrowski's Theorem**.

### A. Ostrowski's Theorem
Ostrowski's Theorem states that every non-trivial absolute value (metric) on the field of rational numbers $\mathbb{Q}$ is topologically equivalent to either the standard real Euclidean absolute value $|\cdot|_{\infty}$ or a $p$-adic absolute value $\|\cdot\|_p$ for some prime number $p$ [2, 3].

This theorem proves that we cannot invent arbitrary new metrics; we must choose between the continuous, leaking Euclidean metric or the discrete, non-leaking $p$-adic metric.

### B. The $p$-adic Valuation
Let $p$ be a prime number. Any non-zero rational number $x \in \mathbb{Q}$ can be written uniquely in the form:

$$x = p^r \frac{a}{b}$$

where $r, a, b$ are integers, and $p$ does not divide $a$ or $b$. We define the $p$-adic valuation $v_p(x)$ as:

$$v_p(x) = r$$

and we define $v_p(0) = \infty$.

### C. The $p$-adic Absolute Value
Using the valuation, we define the $p$-adic absolute value $\|\cdot\|_p$ of $x$ as:

$$\|x\|_p = p^{-v_p(x)}$$

and we define $\|0\|_p = 0$.

Under this metric, a number is "small" if it is divisible by a high power of $p$. This property inverts the classical Euclidean scale, allowing infinite-precision fractional details to be treated as algebraically small, bounded residuals.

## III. The Strong Triangle Inequality (The Ultrametric Property)

The core computational benefit of the $p$-adic absolute value is that it satisfies the **strong triangle inequality** (also known as the ultrametric inequality).

### A. The Strong Triangle Inequality
For any $x, y \in \mathbb{Q}$, the metric satisfies:

$$\|x + y\|_p \le \max\left(\|x\|_p, \; \|y\|_p\right)$$

In an ultrametric space, the distance between two points does not accumulate when they are added. Geometrically, this implies that every point inside a $p$-adic ball is its center, and two balls are either completely disjoint or one is completely contained within the other [3].

### B. Compile-Time Error Containment
We apply this property to the execution of the Cohomological Task Stack. Let $x_z$ and $y_z$ be truncated approximations of two numbers at zoom level $z$, carrying the respective truncation error residuals $e_1$ and $e_2$:

$$\|e_1\|_p \le p^{-z}, \quad \|e_2\|_p \le p^{-z}$$

When we perform addition ($\oplus$) on these registers, the combined error of the sum does not grow:

$$\|e_1 \oplus e_2\|_p \le \max\left(\|e_1\|_p, \; \|e_2\|_p\right) \le p^{-z}$$

Because the error of the sum is bounded by the *maximum* of the individual errors rather than their *sum*, the truncation error **never accumulates** during arithmetic operations. The error remains strictly contained by the truncation depth $z$. This provides the exact mathematical mechanism required to prevent numerical drift and preserve SMR determinism in asynchronous distributed networks.

## IV. Coprime Base Decomposition ($\mathbb{Q}_{10} \cong \mathbb{Q}_2 \times \mathbb{Q}_5$)

To support human-readable decimal inputs (base 10) on standard binary hardware registers (base 2) without introducing representation loss, the $Qn$ primitive performs exact base translations.

### A. The Algebraic Decomposition
The number 10 is composite, meaning the 10-adic absolute value does not define a multiplicative norm on its own. However, under the algebraic properties of profinite rings, the ring of $10$-adic integers $\mathbb{Z}_{10}$ decomposes exactly into the product of the $2$-adic and $5$-adic integers [5, 6]:

$$\mathbb{Z}_{10} \cong \mathbb{Z}_2 \times \mathbb{Z}_5$$

By extending this to their fields of fractions, we obtain the coprime decomposition:

$$\mathbb{Q}_{10} \cong \mathbb{Q}_2 \times \mathbb{Q}_5$$

### B. The Execution Path
When a user inputs a decimal fraction (e.g., $0.1_{10} = 1/10$), the $QnLang$ compiler splits the value into two parallel registers:

$$0.1_{10} \mapsto \left( x_2, \; x_5 \right) \in \mathbb{Q}_2 \times \mathbb{Q}_5$$

The **$2$-adic component** ($x_2$) is executed natively on standard binary CPU registers for maximum processing speed. The **$5$-adic component** ($x_5$) represents the fractional portion that cannot be represented exactly in binary. It is carried algebraically as an exact, symbolic cofactor (the remainder) [5, 6].

This exact decomposition prevents representation drift. When the value is subsequently converted back to base 10 for human verification, the abstraction layer recombines the components, reconstructing the exact decimal value $0.1_{10}$ with **zero information loss**.

## VI. Glossary of Terms for Non-Technical Readers

*   **Ultrametric Space:** A metric space where the distance between points satisfies a "strong" version of the triangle inequality. In this space, distances do not accumulate during addition, and triangles are always either isosceles or equilateral.
*   **$p$-adic Valuation:** A mathematical function that measures the power of a prime number $p$ that divides a given number. It determines the "multiplicity" of $p$ in the number's factorization.
*   **$p$-adic Absolute Value:** A non-classical way of measuring the size of a number. Under this metric, a number is considered small if it is divisible by a large power of $p$.
*   **Ostrowski's Theorem:** A foundational theorem in number theory proving that the only possible ways to measure the distance between rational numbers are the standard real distance (Euclidean) and the prime-power distance ($p$-adic).
*   **Coprime Base Decomposition:** The algebraic process of breaking down a composite number base (like base 10) into its prime, non-overlapping components (like base 2 and base 5) to allow exact, parallel calculations.
