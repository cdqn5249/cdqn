### **Core Goals (Canonical Invariants)**

#### **Goal 1: Sovereign Communication Substrate**
**Invariant**: Bidirectional, verifiable message exchange between independent nodes without centralized infrastructure.

**Validation Environment**: GitHub repository (immutable server) ↔ Termux Android (mobile client).

**Success Criteria**:
- Structured messages flow bidirectionally via git commits and workflow triggers
- GH-Pages provides real-time human-readable documentation of all exchanges
- End-to-end loop completes without manual intervention (commit → documentation → response → verification)

#### **Goal 2: cdqnProt Structured Protocol**
**Invariant**: Cryptographically verifiable communication using the KDU schema over git-based transport.

**Validation Environment**: Same GitHub-Termux funnel with binary KDU files replacing text messages.

**Success Criteria**:
- Binary KDUs serialize/deserialize correctly with exact byte-for-byte fidelity
- Signatures verify across the network boundary
- Teaser extraction works—non-runtime users see curiosity hooks, runtime users access full content
- 10 complete message exchanges validate the full protocol lifecycle

#### **Goal 3: Pure Functional Entity Processing**
**Invariant**: Entities process KDUs through pure mathematical functions that evolve world state immutably.

**Validation Environment**: Termux entities processing GitHub KDUs, with causal chains preserved in journals.

**Success Criteria**:
- Entity behaviors return `(new_world_state, response_kdus)` without hidden mutations
- World state reconstruction from journal snapshots is perfect
- 100 KDU processing cycles complete with verifiable causal evolution

#### **Goal 4: Modular Architecture (C/U/K Layers)**
**Invariant**: Clear separation of concerns with pure functional interfaces between Core infrastructure, User entities, and Kernel truth.

**Validation Environment**: GitHub running C modules (workflows), Termux running U modules (entities), shared K module (KDU factory).

**Success Criteria**:
- K layer enforces cryptographic invariants (signatures, hashes)
- C layer provides infrastructure (persistence, trust) through pure transformations
- U layer contains user logic with modular boundaries
- 5-module pipeline processes KDUs correctly through all layers

#### **Goal 5: Immutable Economic Participation**
**Invariant**: Economic actions (gas, minting, transfers) create explicit, verifiable state evolution without mutable balances.

**Validation Environment**: Sponsorship flow from GitHub to Termux culminates in cdqnStar minting.

**Success Criteria**:
- Gas accounting deducts correctly from originator balance
- Sponsorship minting creates 10 cdqnStar with immutable proof
- Economic state evolution reconstructs perfectly from KDU chain
- Licensing compliance enforced at protocol boundary

#### **Goal 6: Complete Sovereign Onboarding**
**Invariant**: New nodes join the network through verifiable lineage establishment and economic participation.

**Validation Environment**: Full 5-KDU chain: GitHub sponsorship request → Termux OriginKDU → mutual signing → minting → network announcement.

**Success Criteria**:
- OriginKDU chain completes with 5 linked KDUs
- Lineage reconstruction verifies across network boundary
- cdqnStar minting succeeds with balance evolution
- Both nodes confirm full economic and identity state

### **Risk Management**

**Communication Risks**:
- **GitHub Rate Limits**: Exponential backoff in Termux scripts; fallback to manual dispatch
- **Termux Connectivity**: Offline queuing with automatic retry on reconnect
- **Workflow Timeouts**: Modular workflows (one per message type)

**Purity Risks**:
- **State Leaks**: TDD with explicit world state validation
- **Mutation Creep**: Code review checklist for `&mut` usage
- **Complexity**: Maximum 3 transformations per processing stage

**Validation Risks**:
- **False Positives**: Double validation (unit + integration + manual)
- **Environment Flakiness**: Containerized test suite for reproducibility
- **Documentation Drift**: Auto-generate docs from test cases
